今天 / 回看路由拆分 Spec

创建于 2026-05-24。目标是把当前共享的 FlowRoute / FlowViewModel 拆成清晰的“今天”和“回看”边界，避免两个功能继续在 UI 与状态层耦合。

背景

当前底部导航已经没有独立的用户可见 Flow 页，但代码中 FLOW_TODAY 与 FLOW_REVIEW 都进入 FlowRoute。FlowRoute 会收集 FlowViewModel.uiState，再在 FlowScreen 内根据 FlowFocus 分支展示“今天”或“回看”。

这造成两个实际问题：第一，打开“回看”也会依赖“今天”的聚合状态和模型转换；第二，后续修改“今天”的行动中枢、知识压缩、任务发现时，容易影响只需要回看入口和历史会话的页面。

目标

推荐实现方案：保留历史 route 字符串以兼容通知、Widget 和 deep link，但把运行时边界拆成 TodayRoute / TodayViewModel / TodayUiState 与 ReviewHomeRoute / ReviewHomeScreen。

  • 今天 负责行动中枢、今日聚焦、自动发现、跟踪任务、今日回看提示。
  • 回看 负责回看入口、最近会话、推荐问题、进入回看聊天与历史。
  • FlowFocus 只作为旧入口兼容层存在，把旧通知或 intent 映射到 今天 或 回看。

非目标

  • 不重命名持久化 route 字符串，例如 flow/today、flow/review，避免破坏历史 PendingIntent 和深链。
  • 不在本次拆分中重做“今天”的产品视觉或回看聊天能力。
  • 不删除底层 planner、repository 或知识层模型；只调整 UI route 与 ViewModel 边界。
  • 不安装 debug APK，不清空模拟器或真机数据。

架构设计

1. Today 边界。将当前 FlowViewModel 重命名或迁移为 TodayViewModel，将 FlowUiState 改为 TodayUiState。它继续承载今日计数、继续推进记录、旧记录重连、主线候选、知识压缩、自动发现、跟踪任务和今日回看提示需要的状态。

2. Review 边界。新增 ReviewHomeRoute 和 ReviewHomeScreen。它只观察 ReviewChatSavedConversationRepository.observeLatestSavedSessionSummary()，并通过导航回调进入回看聊天、回看历史或指定保存会话。它不依赖 TodayViewModel，也不计算 TodayDesignModel。

3. 共享组件。ReviewChatEntryCard 保持可复用组件身份，但移出 flow 包或至少改为由回看 home 明确拥有。TodayReviewHintCard 仍可在“今天”中展示最近回看提示，但只接收一个轻量的 SavedReviewChatSessionSummary。

4. 兼容适配。MindFlowDestinations 可继续保留 FLOW_TODAY、FLOW_REVIEW 常量名作为兼容层，但导航 graph 中分别接入 TodayRoute 和 ReviewHomeRoute。如果改名为 TODAY / REVIEW，必须保留旧字符串值。

迁移步骤

  1. 新增 ReviewHomeRoute，让 FLOW_REVIEW 先改走独立 route，并删除回看路径对 FlowViewModel 的依赖。
  2. 将 FlowRoute 拆为 TodayRoute，删除 FlowPage、FlowFocus.toPage() 与回看分支。
  3. 将 FlowViewModel / FlowUiState 迁移为 TodayViewModel / TodayUiState，同步更新 TodayDesignModel 和 TodaySecondaryScreens。
  4. 清理确定无引用的私有 UI 函数、测试名称和导入；保留历史 route 字符串与旧入口映射。
  5. 更新 PROJECT_GUIDE.md 和 implementation-notes.html，说明拆分后的边界。

验收标准

  • 点击底部 今天 时，只创建/收集 TodayViewModel 相关状态。
  • 点击底部 回看 时，不依赖 TodayViewModel，不计算 TodayDesignModel。
  • 代码中不再存在可直接调用的 FlowRoute 或 FlowViewModel 类型；保留的 FLOW_TODAY / FLOW_REVIEW 只作为历史 route 字符串兼容层。
  • 通知、Widget 和旧 ACTION_OPEN_FLOW 仍能正确打开“今天”或“回看”。
  • 回看首页仍能打开新聊天、推荐问题、历史列表和最近保存会话。
  • 今天首页、自动发现、今日任务详情仍保持现有行为。

测试与验证

  • 新增或调整路由单测，覆盖旧 FlowFocus.TODAY / FlowFocus.REVIEW 的兼容映射。
  • 新增 Review home 测试，确认它只依赖保存会话摘要和导航回调。
  • 更新 Today 相关单测，将 FlowUiState 引用替换为 TodayUiState。
  • 运行 ./gradlew --no-daemon :app:testDebugUnitTest 与 ./gradlew --no-daemon :app:assembleRelease。
  • 用户可见验证必须安装 release APK app/build/outputs/apk/release/app-release.apk；优先模拟器验证，真机安装前不清数据、不卸载。

风险与取舍

本次拆分会触及导航 graph、Compose route、ViewModel factory 和 Today 测试。最大风险是误删历史入口兼容逻辑，因此 route 字符串和 ACTION_OPEN_FLOW 不应在第一轮删除。更深层的 planner/use case 拆分暂不纳入，避免一次性重构过大。

自我审视结论

补充审视后，我认为 spec 需要明确一个硬性边界：回看 不只是 UI 分支独立，而是运行时不应触发今日状态聚合。因此验收标准增加了“删除可调用的 FlowRoute / FlowViewModel 类型”这一条。这样后续实现不能只在 MindFlowApp 改两个函数名，而必须让回看 route 的参数列表、状态收集和模型转换都脱离今天侧。
